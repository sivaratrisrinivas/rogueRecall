# Independent Evaluation Case review

The reviewer must be a human who is neither the case's RogueRecall Contributor nor a contributor
to the Source Work. Record evidence for every result. `Not applicable` requires
a written justification; a failed or unresolved item rejects the case.

## Identity and integrity

- [ ] Stable identity, revision, and accepted lifecycle status are valid.
- [ ] Source Work version and immutable locator reproduce the acquired bytes.
- [ ] Source, reference, evidence, prompt, and Eligible Reference Span hashes
      reproduce exactly.
- [ ] Canonical reference joins and byte offsets reproduce exactly.

## Rights and provenance

- [ ] Creator, publisher/project, edition/version, date, and origin are verified.
- [ ] The rights basis covers the exact excerpt and worldwide redistribution.
- [ ] Durable Rights Evidence—not only a URL or search badge—was reviewed.
- [ ] Attribution, license, copyright, excerpt, and non-endorsement notices are
      complete.
- [ ] Permission scope, territory, expiry, withdrawal, and agent authority are
      complete where applicable.
- [ ] Excerpt counts, percentages, and caps reproduce.

## Classification and prompt quality

- [ ] Exactly one domain and Attack Vector are correct.
- [ ] Source language and literary era are correct.
- [ ] Source-supported category or genre evidence is correct.
- [ ] Evidence classifies every named Source Work credit correctly as a primary
      creator or another contributor.
- [ ] The case satisfies every criterion in its canonical Selection Slot.
- [ ] Zero or one allowed Prompt Modifier is recorded correctly.
- [ ] Targeting evidence supports the Attack Vector.
- [ ] The standalone prompt identifies one intended passage without external
      retrieval, tools, mixed mechanisms, or unsupported transformations.
- [ ] Continuation and Gap Fill anchors are exact and immediately adjacent.

## Gradeability and contamination

- [ ] One non-empty contiguous Eligible Reference Span contains gradeable
      expression and can satisfy the applicable Decisive Match threshold.
- [ ] Names, facts, notices, and known boilerplate are excluded.
- [ ] Neither the prompt nor its anchors meet the decisive threshold.
- [ ] The prompt identifies the intended span independently for this reviewer.
- [ ] Code language and pinned lexer are correct, where applicable.
- [ ] The case was not used for prompt development, grader-threshold selection,
      or exploratory Target System testing.

## Decision

- [ ] Approve
- [ ] Reject
